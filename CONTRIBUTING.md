## Getting Started

### 1. Fork the Repository

Click the **Fork** button at the top right of the [Reflex repository](https://github.com/Codimow/Reflex).

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/Reflex.git
cd Reflex
```

### 3. Create a Branch

Create a branch for your feature or fix:

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

**Branch naming conventions:**
- `feature/` â€” New features
- `fix/` â€” Bug fixes
- `docs/` â€” Documentation changes
- `refactor/` â€” Code refactoring

### 4. Make Your Changes

Write your code and make sure to:

- Follow the existing code style
- Add tests for new functionality
- Update documentation if needed

### 5. Test Your Changes

```bash
go test ./...
```

### 6. Commit Your Changes

Write clear, concise commit messages:

```bash
git add .
git commit -m "feat: add custom delay flag"
```

**Commit message format:**
- `feat:` â€” New feature
- `fix:` â€” Bug fix
- `docs:` â€” Documentation
- `test:` â€” Tests
- `refactor:` â€” Code refactoring

### 7. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 8. Open a Pull Request

1. Go to the original [Reflex repository](https://github.com/Codimow/Reflex)
2. Click **New Pull Request**
3. Select your fork and branch
4. Fill out the PR template
5. Submit!

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the code, not the person

## Questions?

Open an issue or start a discussion. We're happy to help!
